# 프로젝트: 용병단 마을 (가제)

> “용병단을 키워 던전에 보내 재화를 벌고, 그 재화로 마을과 용병단을 성장시키는 2D 오토배틀러.”
>

---

## 0. 디자인 방향 / 철학

- **난이도 지향**
    - 전반적으로 **라이트~중간 난이도**
    - 패배 시 페널티는 있지만, 진행이 막힐 정도로 과도하지 않게 설계
    - 추천 전투력보다 낮게 도전하면 도전/트라이 느낌, 맞추면 안정적으로 클리어
- **진행 구조**
    - **챕터형 메인 캠페인 + 엔드리스 파밍 스테이지**
    - 챕터별로 구간 목표(보스/지역 클리어)를 두고, 이후 구간에선 반복 파밍이 가능한 스테이지를 제공
- **용병 사망/리스크**
    - 기본적으로 **영구 사망은 없음**
    - 전투에서 쓰러진 용병은 성당/사원에서 **골드(또는 자원)를 지불해 부활**
- **방치/시간 기반 시스템**
    - 초기 버전(v1)에서는 **방치/오프라인 수입 없음**
    - 모든 주요 재화는 **던전 플레이를 통해서만 획득**
- **타겟 플레이 패턴**
    - 싱글 플레이 기준, **하루 30분~1시간 정도 가볍게 던전 몇 개 돌리며 파밍하는 게임**
    - 깊게 파고들면 빌드/조합 고민 요소가 있지만, 기본 흐름은 부담 없이 즐기는 방향

---

## 1. 게임 개요

- **장르**
    - 2D 오토배틀러 + 라이트 타운 빌딩 + 파밍 RPG
- **플랫폼**
    - PC
- **플레이 타임**
    - 1회 세션: 10~30분
    - 전체 진행: 수십 시간 이상 파밍 가능
- **카메라/그래픽**
    - 2D 탑다운 뷰 전투
    - 마을은 2D 배경 위에 건물 몇 개가 보이는 허브 화면 (실제 기능은 메뉴형)

---

## 2. 핵심 컨셉 & 재미 포인트

### 2.1 핵심 판타지

1. **용병단을 키운다**
    - 다양한 용병을 모집하고, 성장시키고, 조합을 맞추는 재미
2. **던전에서 돈을 벌어온다**
    - 전투는 자동으로 진행되고, 플레이어는 편성/전략에 집중
3. **마을이 성장한다**
    - 허름한 마을 → 건물 업그레이드 → 새로운 기능/용병 해금

### 2.2 플레이어가 느끼는 재미

- 던전이 점점 덜 힘들어지는 **성장감**
- 던전 파밍 → 마을 강화 → 더 강한 용병 확보 → 다시 파밍하는 **성장 루프**
- “이번에는 이 조합으로 가볼까?” 하는 **빌드 실험**
- 건물 레벨업에 따른 비주얼/효과 변화로 느끼는 **눈에 보이는 발전**

---

## 3. 코어 게임 루프

1. **마을 화면**
    - 용병 관리 / 마을 건물 업그레이드
2. **던전 선택**
    - 지역 / 층 / 난이도 선택
3. **전투 준비**
    - 출전 용병 선택
    - 초기 위치(배치) 설정
4. **자동 전투 진행**
5. **결과 / 보상 획득**
    - 골드, 재료, 장비 등
6. 다시 마을로 돌아가 1~5 반복

> 루프 구조: Town → Dungeon → Battle → Reward → Town
>

---

## 4. 시스템 큰 틀

### 4.1 용병 시스템 (Mercenaries)

**역할(Role) 기본 3~4종**

- **탱커**: 앞열, 높은 HP/방어, 낮은 공격력
- **딜러(근접/원거리)**: 높은 공격력, 낮은 방어
- **서포터/힐러**: 회복, 버프, 디버프 담당 (초기엔 힐러 1종만 있어도 OK)

**성장 방식(초기 버전)**

- **레벨 업**
    - 전투 보상(골드)로 레벨업
- **희귀도 / 승급** (나중 도입)
    - 같은 용병 여러 개 합쳐 ★ 업 (예: 같은 카드 3장 → ★2)
- **장비 장착**
    - 무기/방어구 등 1~2슬롯으로 최소화

---

### 4.2 전투 시스템 (Battle)

- **전개 방식**
    - 2D 탑다운 뷰, 하나의 방/전장 위에서 아군/적군이 함께 배치
    - 카메라는 전장을 통째로 내려다보는 고정 시점
- **전장 구조**
    - 직사각형 맵(작은 방) 기반
    - 장애물은 v0.1에서는 없음, 나중에 기둥/벽 등 추가 여지
- **배치**
    - 전투 시작 전, 플레이어가 아군 용병을 시작 지점 여러 슬롯 중에 배치
    - 근접/탱커는 전방, 원거리/서포터는 후방 쪽에 두는 것이 기본 전략
- **행동 규칙 (기본 AI)**
    - 각 유닛은 가장 가까운 적에게 자동으로 이동
    - 사거리 안에 들어오면 자동 공격
    - 스킬은 쿨타임에 따라 자동 발동(초기에는 패시브/자동 스킬 위주)
- **목표 및 판정**
    - 적을 모두 처치하면 승리
    - 아군이 전멸하면 패배
    - 승리: 골드/재료/장비 획득
    - 패배: 일부 보상 감소, 혹은 소모품/시간만 손해 (구체 규칙은 추후)

> 목표: 조작은 단순하지만, 배치/조합/스탯 설계로 승패가 갈리는 탑다운 오토 전투
>

---

### 4.3 마을 시스템 (Town)

**철학:**

> "마을은 예쁜 메뉴 모음" → 건물 클릭 = 해당 시스템으로 이동.
>

**기본 건물 (v1 기준 4개)**

1. **길드 하우스**
    - 보유 용병 목록 확인
    - 용병 레벨업 / 해고
    - 길드 레벨 업: 최대 용병 보유 수 증가, 전체 스탯 패시브
2. **술집(주점)**
    - 신규 용병 모집(랜덤 라인업)
    - 갱신 버튼으로 라인업 변경(골드 소모 등)
    - 건물 레벨업 시: 슬롯 수 증가, 고레어 용병 등장 확률 증가
3. **대장간**
    - 장비 강화(초기에는 강화 위주)
    - 건물 레벨업 시: 강화 상한 증가, 추가 옵션 해금
4. **성당/사원**
    - 전투에서 쓰러진 용병 부활
    - 건물 레벨업 시: 부활 비용 감소, 자동 부활 확률 상승 등

> 마을 배경은 이 4개 건물이 고정된 단일 화면으로 시작.
>
>
> 각 건물은 실질적으로는 UI 메뉴 하나.
>

### 4.4 던전 시스템 (Dungeon)

**구조**

- **지역(챕터) 단위**
    - 예: 초원 → 숲 → 동굴 → 화산…
- 각 지역은 여러 **스테이지**로 구성
    - 스테이지 클리어 → 다음 스테이지 해금
- 반복 파밍용 스테이지 존재
    - 예: [초원 3-5]는 골드 파밍 전용

**난이도**

- 각 스테이지에 **추천 전투력** 표기
- 일반 / 엘리트 / 보스 스테이지로 구분 가능

**보상 구조**

- 기본: 골드
- 확률: 장비, 재료
- 첫 클리어 보상: 추가 골드 or 용병/장비 등 특별 보상

---

### 4.5 경제 시스템 (Economy)

**최초 자원 구성 (최소화)**

1. **골드**
    - 대표 재화
    - 용병 고용 / 레벨업 / 장비 강화 / 건물 업그레이드 등에 사용
2. **재료(1~2종)**
    - 대장간에서 장비 강화/제작에 사용
    - 특정 던전에서 주로 드랍

> 상업화용 프리미엄 재화는 지금 단계에선 제외.
>

## 5. 최소 스코프 버전 (MVP: v0.1)

“이 기능들만 있어도 ‘게임 같다’ 싶은 최소 세트”

### 전투

- 아군 최대 4명 / 적 최대 4명
- 작은 직사각형 탑다운 전장(방)에서 진행
- 전투 시작 전 아군 시작 위치(슬롯 배치)만 설정
- 스탯: HP, ATK, ATK SPD, RANGE 정도
- 자동 전투 + 승/패 판정

### 용병

- 역할 3종 (탱커 / 근접딜러 / 원거리딜러)
- 레벨업만 존재, 희귀도/합성은 후순위
- 술집에서 골드로 고용

### 마을

- 마을 배경 1장
- 건물 2개부터 시작: 길드 하우스, 술집
- v0.1에서는 건물 레벨 고정(업그레이드 없음)으로 시작 가능

### 던전

- 지역 1개 (예: 초원)
- 스테이지 5~10개
- 스테이지별 적 구성/난이도만 다르게 설정

### 경제

- 자원: 골드 1종
- 던전 클리어 → 골드 획득
- 골드로 용병 고용/레벨업

---

## 6. 이후 확장 아이디어 (메모)

- 용병 희귀도/합성 시스템 (★1 → ★2 → ★3)
- 직업/종족 시너지 시스템 (예: 전사 3명 → 방어력 증가)
- 마을 건물 추가 (마법탑, 시장, 훈련소 등)
- 장비 옵션 랜덤 생성(로그라이크 느낌)
- 자동 반복 전투 / 파밍 기능
- 간단한 스토리/세계관, 퀘스트, NPC 대사 등

---

## 7. 개발 우선순위 (러프)

1. 전투 코어 구현
    - 유닛 이동/공격/사망까지 돌아가는 전투 루프
2. 던전 → 전투 → 보상 루프
    - 스테이지 선택 → 전투 → 골드 획득
3. 용병 관리 + 술집(상점)
    - 용병 목록, 출전 세팅, 구매
4. 마을 화면
    - 배경 + 건물 버튼 UI
5. 건물 업그레이드 / 추가 자원 / 장비 시스템 확장
